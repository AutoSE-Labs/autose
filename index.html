THE WORKFLOW

One request flows through the whole lifecycle.

Typed artifacts pass between phases. Quality gates guard the transitions. You approve before anything is written.

  1. PH-01

    Requirements

    Plain-English intent is parsed into a structured requirement.

  2. PH-02

    Architecture

    AutoSE plans structure and reasons about trade-offs and constraints.

  3. PH-03

    ADRs

    Decisions are recorded, linked, and proposed for review.

  4. PH-04

    Generation

    Complete code files are produced against your standards.

  5. GATE

    Your review

    Inspect, diff, approve, or reject. Nothing writes without sign-off.

CAPABILITIES

Engineering you can account for.

SOVEREIGNTY

Runs on what you own

All inference goes through Ollama on your machine or an org-hosted endpoint. It calls no third-party APIs and sends no telemetry.

CONTEXT

Grounded in your org

A profile encodes your coding standards, approved libraries, and architectural rules. Every output is shaped by them.

GOVERNANCE

Traceable by default

Requirements link to decisions, decisions to code, and code to review. The audit trail is part of how the runtime works.

CONTROL

You hold the pen

Human review is a built-in step. Agents prepare and propose. You approve, refine, or reject.

ROUTING

Right model per task

Work is routed between smaller and larger local models by complexity, which keeps quality high while controlling speed and cost.

SURFACE

CLI or VS Code

Drive workflows from an interactive terminal session, or from a sidebar panel inside your editor.

GET STARTED

Three commands to your first workflow.

Most people just want the app. Download the desktop installer and you are set; the steps below are for the terminal version.

Download the Windows app (.exe) macOS and Linux coming soon.

  1. 1
    Install the CLI
    Run the one-line command for your OS (in the panel). It clones AutoSE, sets up its environment with uv, and adds the autose launcher to your PATH.
  2. 2
    Set up a local model
    AutoSE runs on inference you own. Install Ollama and pull a coder model: ollama pull qwen2.5-coder:14b
  3. 3
    Run it in your project
    From any project folder, run autose for an interactive session, or autose "add tests for the parser" to start from a prompt.
